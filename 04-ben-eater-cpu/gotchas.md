# Ben Eater 8-bit CPU — Gotchas

## 1. Power Distribution Nightmares

**Problem:** 14 breadboards draw significant current. Voltage drops across breadboard power rails.

**Symptoms:** ICs behave erratically, LEDs dim, intermittent failures.

**Solutions:**
- Use thick 22AWG wire for power distribution (NOT thin jumper wires)
- Distribute power to each breadboard independently (star topology, not daisy-chain)
- Add 0.1µF decoupling capacitor across VCC-GND of EVERY IC
- Add 10-100µF bulk capacitor at power entry point
- Measure voltage at furthest breadboard — should be 4.75V-5.25V

**Absolute minimum:** 5V 2A regulated power supply. USB phone charger may NOT be enough.

## 2. Floating Inputs — The Silent Killer

**Problem:** TTL inputs float HIGH when disconnected, but unreliably. Can pick up noise.

**Rule:** NEVER leave a TTL input floating. Tie unused inputs to VCC (logic HIGH) or GND (logic LOW) via 10kΩ resistor.

**Common failure:** 74LS138 enable pins, 74LS283 carry-in, 74LS157 select pins left floating → random behavior.

## 3. LED Current Limiting — Don't Skip

**Problem:** Connecting LED directly to TTL output → excessive current → IC damage or output voltage collapse.

**Solution:** 220Ω-470Ω resistor in series with EVERY LED.

**Gotcha:** Blue/white LEDs need ~3.3V forward voltage. TTL output HIGH is only ~3.4V at low current. Blue LEDs may not light without buffer/driver.

## 4. 74LS vs 74HC Compatibility

**Problem:** 74LS (TTL) and 74HC (CMOS) have different voltage thresholds.

| Parameter | 74LS (TTL) | 74HC (CMOS) |
|-----------|-----------|-------------|
| V_IH (min input HIGH) | 2.0V | 3.5V (at 5V VCC) |
| V_OH (min output HIGH) | 2.7V | 4.9V |
| V_IL (max input LOW) | 0.8V | 1.5V |

**Gotcha:** 74LS output HIGH (2.7V) may NOT register as HIGH on 74HC input (needs 3.5V) → use pull-up resistors or stick to one family.

## 5. 555 Timer Stability

**Problem:** 555 timer jitter, double-triggering, or not oscillating.

**Common fixes:**
- Pin 5 (control voltage): add 0.01µF capacitor to GND (critical for stability)
- Pin 4 (reset): tie to VCC if unused (NEVER float)
- Use 1% tolerance resistors and film capacitors for timing
- Keep timing capacitor leads short

**Debouncing:** Mechanical switch directly connected to 555 → multiple edges. Use RC filter + Schmitt trigger or dedicated debounce circuit.

## 6. EEPROM Programming Gotchas

**Problem:** 28C16 EEPROM requires specific programming sequence.

**Requirements:**
- VCC = 5V (stable!)
- Write pulse: WE low for 1ms minimum (check datasheet — some need 10ms)
- Address must be stable before WE goes low
- Data must be stable before WE goes high
- After write: poll I/O7 for completion or wait 10ms

**Gotcha:** Writing EEPROM while Arduino shares power with breadboard → voltage sag during write → corrupted data. Use separate power or large capacitor.

## 7. Bus Contention — THE Most Common Bug

**Problem:** Multiple 74LS245 transceivers driving the bus simultaneously → short circuit.

**The Golden Rule:** EXACTLY ONE device can output to the bus at any time. Control logic MUST enforce this.

**How to detect:** ICs getting hot, power supply current spike, LEDs dimming.

**Prevention:** Design control logic so bus output enables (AO, CO, EO, RO, IO) are mutually exclusive.

## 8. 74LS173 Register Load Timing

**Problem:** 74LS173 loads on rising clock edge, but data must be stable during setup time.

**Setup time:** ~20ns for 74LS. At low clock speeds (kHz), this is trivial. The real issue is clock skew.

**Gotcha:** If clock arrives at register BEFORE data from RAM/ALU settles → wrong value latched. Use the falling edge of the clock for control signal changes, rising edge for register loads.

## 9. 7-Segment Display Multiplexing

**Problem:** 4 digits × 7 segments = 28 pins. Too many to drive directly.

**Solution:** Multiplexing — light one digit at a time, cycle fast enough for persistence of vision.

**Gotcha:** 74LS273 can drive common-cathode 7-segment display directly, but check current per segment (~8mA per output). Use transistor drivers for common-anode displays.

## 10. Wire Management

**Problem:** 14 breadboards + hundreds of wires = debugging nightmare.

**Best practices:**
- Color code: Red=VCC, Black=GND, Yellow=Clock, Blue=Data bus, Green=Control
- Cut wires to exact length (no loops or spaghetti)
- Route wires at right angles along breadboard grooves
- Label each breadboard and IC
- Take photos after each module completion

**If it looks like a mess, it WILL be a debugging nightmare.**

## 11. Clock Skew and Glitches

**Problem:** Control signals are generated by EEPROM. EEPROM access time is 150-250ns.

**At TTL speeds:** Control signal may glitch during EEPROM address transition.

**Fix:** Use the inverted clock to latch control signals, or ensure address changes happen on falling edge while control signals are sampled on rising edge.

## 12. The 74189 RAM Limitation

**Problem:** 74189 is 16×4 bits = 16 nibbles. You need two for 8 bits = 16 bytes total.

**This is tiny.** Your programs must fit in 16 bytes. This forces elegant, compact code.

**Gotcha:** 74189 outputs are INVERTED (active LOW). The data read from RAM is the complement of what was written. Fix in control logic or with inverters on data bus.

## 13. Running at Full Speed

**Problem:** Works perfectly in single-step mode, fails at full speed.

**Causes:**
- Timing violations (setup/hold times)
- Glitches on control lines
- Power supply noise at higher current draw
- EEPROM access time too slow for clock period

**Fix:** Add 0.1µF decoupling caps to EVERY IC, check EEPROM speed grade, slow down clock.
